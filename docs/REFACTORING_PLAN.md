# План стабилизации и развития

## Уже сделано

- Инициализирован git-репозиторий и создан первый коммит.
- Усилен `.gitignore`: секреты, отчеты, аудио, профили браузера, кэши и локальные БД не попадают в git.
- Устаревшие служебные `.txt` перенесены из корня в `docs/_archive/`.
- Справочные документы перенесены в `docs/`.

## Ближайшие шаги

1. Добавить минимальный `pyproject.toml` с настройками `ruff`, `black` и `pytest`.
2. Вынести структурированные логи в `reports/logs/` через `logging` вместо ручных `print`.
3. Добавить smoke-тесты на ключевую KPI-логику: чек-лист звонка, возражения, CRM-чек-лист, итоговую оценку.
4. Сделать локальную папку аудио основным источником записей, а браузерный UI-fallback оставить только последним вариантом.
5. Начать декомпозицию `pipelines/bitnewton_sync.py` без изменения поведения: сначала вынести scoring/reporting в отдельные модули.

## Среднесрочные улучшения

- SQLite `state.db` для идемпотентности: статус обработки звонка, hash расшифровки, факт прикрепления в Bitrix.
- Валидация `kpi_config*.json` через схему.
- Отдельные настройки профилей клиента: домен, webhook, KPI, Bit.Newton token.
- Дашборд в Streamlit поверх сохраненного состояния, а не только поверх последнего Excel.
- Метрики надежности: доля REST-скачивания, доля UI-fallback, ошибки Bitrix API, время Bit.Newton ASR.

## Стратегия

- Постепенно перейти от ручного запуска Streamlit/CLI к фоновому воркеру.
- Хранить первичные результаты в SQLite, а JSON/XLSX использовать как экспорт.
- Оставить Bit.Newton основным ASR, но держать интерфейс `ASRClient` для будущего резервного провайдера.
